# 프롬프트 엔지니어링 완전 가이드

## 1. 프롬프트 엔지니어링이란?

프롬프트 엔지니어링은 AI 모델에서 원하는 결과를 얻기 위해 입력(프롬프트)을 설계하고 최적화하는 기술입니다. 효과적인 프롬프트는 모델의 응답 품질, 정확성, 관련성을 크게 향상시킬 수 있습니다.

## 2. 핵심 원칙

### 2.1 명확성과 구체성

프롬프트 엔지니어링의 모범 사례는 구체적이고, 명확한 컨텍스트를 제공하며, 예시와 데이터를 포함하고, 원하는 출력을 정의하며, 피해야 할 것보다 해야 할 것에 대한 지시를 제공하는 데 초점을 맞춥니다.

**좋은 예:**
```
다음 고객 리뷰를 분석하여 감정(긍정/부정/중립)과
주요 키워드 3개를 JSON 형식으로 반환해주세요.

리뷰: "이 제품은 정말 훌륭합니다. 배송도 빠르고 품질도 좋아요."
```

**나쁜 예:**
```
이 리뷰 분석해줘
```

### 2.2 구조화된 프롬프트

명확한 구조와 컨텍스트가 세련된 문구보다 더 중요합니다 - 대부분의 프롬프트 실패는 모델의 한계가 아닌 모호함에서 비롯됩니다.

구분자(delimiter)를 사용하면 모델이 프롬프트의 구별되는 부분을 더 잘 이해할 수 있습니다.

```
### 지시사항 ###
아래 텍스트를 한국어로 번역하세요.

### 텍스트 ###
"""
Hello, how are you today?
"""

### 출력 형식 ###
번역 결과만 출력
```

## 3. 주요 기법

### 3.1 Few-shot 프롬프팅

예시를 제공하여 모델이 패턴을 학습하도록 합니다.

```
다음 형식으로 이메일 제목을 분류하세요:

이메일: "50% 할인 세일 - 오늘만!"
분류: 마케팅

이메일: "회의 일정 변경 안내"
분류: 업무

이메일: "비밀번호 재설정 요청"
분류: 보안

이메일: "새로운 기능 출시 안내"
분류:
```

### 3.2 Chain-of-Thought (CoT) 프롬프팅

모델이 단계별로 추론하도록 유도합니다. 이 기법은 모델이 작업 과정을 보여주도록 강제하여 오류를 쉽게 발견할 수 있게 합니다.

```
다음 문제를 단계별로 풀어주세요:

문제: 한 상점에서 사과 5개를 3000원에 샀습니다.
이 사과를 개당 800원에 팔면 총 이익은 얼마인가요?

단계별로 생각해봅시다:
1. 먼저 사과 1개의 원가를 계산합니다
2. 그 다음 1개당 이익을 계산합니다
3. 마지막으로 총 이익을 계산합니다
```

### 3.3 역할 부여 (Role Prompting)

모델에 특정 역할이나 페르소나를 부여합니다.

```
당신은 10년 경력의 시니어 Python 개발자입니다.
코드 리뷰를 할 때 다음 사항을 확인합니다:
- 코드 가독성
- 성능 최적화 가능성
- 잠재적 버그
- 보안 취약점

다음 코드를 리뷰해주세요:
[코드]
```

### 3.4 출력 형식 지정

원하는 출력 형식을 명시적으로 지정합니다.

```
다음 제품 설명을 분석하여 아래 JSON 형식으로 반환하세요:

{
    "product_name": "제품명",
    "category": "카테고리",
    "key_features": ["특징1", "특징2", "특징3"],
    "target_audience": "타겟 고객",
    "price_range": "가격대"
}
```

## 4. 고급 기법

### 4.1 스캐폴딩 (Scaffolding)

복잡한 작업을 작은 단계로 분해합니다.

큰 작업을 작고 구별된 청크로 분해하는 것은 여전히 유용한 기법입니다 - 컨텍스트 제한 때문이 아니라, 모델이 매우 구체적인 요구사항과 범위 내에서 최선의 작업을 수행하는 데 집중할 수 있도록 도와주기 때문입니다. 명확한 경계가 있는 집중된 작업은 단일 프롬프트에서 여러 목표를 달성하려는 것보다 일관되게 더 높은 품질의 결과를 생성합니다.

### 4.2 앵커링 (Anchoring)

관련 정보나 컨텍스트를 먼저 제공하여 모델의 응답 방향을 설정합니다.

### 4.3 적대적 테스트 (Adversarial Testing)

프롬프트의 취약점을 테스트하고 개선합니다.

## 5. 모델 파라미터

### Temperature

Temperature는 모델이 덜 가능성 있는 토큰을 얼마나 자주 출력하는지의 척도입니다 - temperature가 높을수록 출력이 더 무작위적(그리고 보통 더 창의적)입니다.

- **Temperature 0**: 데이터 추출, 사실 기반 Q&A 등 사실적 사용 사례에 적합
- **Temperature 0.7-1.0**: 창의적 글쓰기, 브레인스토밍에 적합

## 6. 피해야 할 실수

### 과도한 엔지니어링
가장 좋은 프롬프트는 가장 길거나 복잡한 것이 아닙니다. 최소한의 필요한 구조로 목표를 안정적으로 달성하는 것입니다.

### 기본 무시
고급 기법은 핵심 프롬프트가 불명확하거나 모호하면 도움이 되지 않습니다.

### AI가 마음을 읽는다고 가정
원하는 것에 대해 구체적으로 설명하세요. 모호하게 남겨두면 AI가 잘못 해석할 여지가 생깁니다.

## 7. 할루시네이션 줄이기

관련 데이터나 컨텍스트를 제공하면 모델이 작업할 사실적 기반을 갖게 되고, 모델에게 "모르겠습니다"라고 말할 수 있는 명시적 권한을 부여하면 조작된 답변의 가능성이 줄어듭니다.

```
다음 질문에 답하세요. 제공된 컨텍스트에서 답을 찾을 수 없으면
"제공된 정보에서 답을 찾을 수 없습니다"라고 답하세요.

컨텍스트: [관련 정보]

질문: [사용자 질문]
```

## 8. 현대적 고려사항 (2024-2025)

이전 AI 모델에서 인기 있었던 일부 프롬프트 엔지니어링 기법은 Claude와 같은 최신 모델에서는 덜 필요합니다.

최신 모델은:
- 더 긴 컨텍스트를 처리할 수 있음
- 모호한 지시도 더 잘 이해함
- 복잡한 추론을 자체적으로 수행할 수 있음

그러나 명확성과 구체성은 여전히 중요합니다.

## 9. 반복과 테스트

프롬프트를 반복적으로 테스트하세요. 문구, 구조, 또는 지시 순서의 작은 변화가 출력을 변경시키며, 시도해봐야만 모델이 어떻게 반응하는지 알 수 있습니다.

다양한 프롬프트 구조를 실험하고 출력 품질에 따라 개선하세요.

## 10. 결론

효과적인 프롬프트 엔지니어링은 명확한 커뮤니케이션, 구조화된 접근 방식, 그리고 지속적인 반복 개선의 조합입니다. 기술적 기법도 중요하지만, 가장 기본적인 원칙은 명확하고 구체적인 지시를 제공하는 것입니다.
